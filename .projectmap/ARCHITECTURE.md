## Overview
ResearchAI (plugin name `research-assistant`) is a Claude Code plugin for Stats/ML
researchers whose core output is **developing new algorithms**. It bundles three
independently-usable skills ‚Äî `literature-explorer` (`/explore`), `algo-brainstorm`
(`/algo`), and `paper-writer` (`/write`) ‚Äî plus a `research-conductor` autopilot
(`/research`) that sequences the other three from idea to paper by reading the
shared `stage` machine. They share state across sessions via
`.research-state/<topic-slug>.md`. The plugin is overwhelmingly prompt-driven:
behaviour lives in Markdown (per-skill `SKILL.md`, per-mode files, shared prompts,
venue/style profiles); the indexed Python is just the deterministic glue ‚Äî literature
retrieval, citation auditing, figures, and multi-model council dispatch.

## Entry points
User-facing entry points are four slash commands (`commands/research.md`,
`commands/explore.md`, `commands/algo.md`, `commands/write.md`) that load the
matching `SKILL.md`. `/research` loads `skills/research-conductor/SKILL.md`, a
prompt-only orchestration loop (`routing.md` + `bootstrap.md`, no Python) that
drives the other three. The runnable Python/shell scripts the skills invoke:
- `shared/council.py` ‚Äî multi-model `--council` panel dispatcher (CLI; stdlib-only)
- `skills/literature-explorer/scripts/search_{arxiv,openalex,semantic_scholar}.py` ‚Äî retrieval CLIs
- `skills/paper-writer/scripts/verify_citations.py` ‚Äî citation auditor CLI
- `skills/paper-writer/scripts/build_paper.sh` ‚Äî LaTeX compile gate + DOCX export (not in `tags`)
- `skills/paper-writer/scripts/figs.py` ‚Äî figure-style library; also a standalone smoketest

## Conventions
- **Map scope ‚Äî read this first:** ctags only indexed Python, but the substance of
  this repo is Markdown. Grep `commands/*.md`, `skills/*/SKILL.md`, `skills/*/modes/*.md`,
  `skills/*/checklists/*.md`, and `shared/prompts/*.md` directly ‚Äî they are **not** in `tags`.
- **Skills are mode-based:** a skill dispatches to one `modes/<name>.md` per invocation
  (e.g. `/algo gap-analysis`, `/write outline`). Add behaviour as a new mode file.
- **Cross-skill/session state** lives in `.research-state/<topic-slug>.md`, per
  `shared/research_state.schema.md` ‚Äî the handoff between explore ‚Üí algo ‚Üí write.
- **Python:** stdlib + a few deps (`httpx`, `tenacity`, `pybtex`; `matplotlib`/`numpy`
  optional via the `figures` extra). Managed by uv (`uv sync`), `package = false`.
  Retrieval/audit scripts use `tenacity` retries against external APIs.
- **MIT-licensed throughout; no copyleft or non-commercial deps** (project is license-sensitive).
- **Anti-sycophancy / anti-hallucination** are baked into the prompts: brainstorm responses
  must surface weaknesses first, and `--council` output is ideation-only ‚Äî its citations are
  `[VERIFY]`-flagged and must pass the citation audit before entering research-state.
